# Contributing

## Setup

```bash
make setup    # Install pre-commit hook + dev dependencies
```

## Adding a Plugin

1. Create a directory under `plugins/<your-plugin>/`
2. Add `.claude-plugin/plugin.json` (Claude Code manifest):

   ```json
   {
     "name": "your-plugin",
     "description": "What it does",
     "version": "1.2.2",
     "author": { "name": "Your Name" }
   }
   ```

3. Add `.codex-plugin/plugin.json` (Codex CLI manifest):

   ```json
   {
     "name": "your-plugin",
     "version": "1.2.2",
     "description": "What it does",
     "skills": "./skills-codex/",
     "interface": {
       "displayName": "Your Plugin",
       "shortDescription": "What it does",
       "developerName": "Your Name",
       "category": "Development"
     }
   }
   ```

4. Add `gemini-extension.json` (Gemini CLI manifest):

   ```json
   {
     "name": "your-plugin",
     "version": "1.2.2",
     "description": "What it does"
   }
   ```

5. Add skills, agents, hooks, or commands in their respective directories
6. Register the plugin in both marketplaces:
   - `.claude-plugin/marketplace.json` (Claude Code)
   - `.agents/plugins/marketplace.json` (Codex CLI)
7. Run `make build` to regenerate every derived artifact

## Available Commands

```bash
make help             # Show all commands
make ci               # Run full CI pipeline locally (lint + validate + check + test)
make lint             # Run all linters
make test             # Run pytest
make validate         # Validate canonical sources (frontmatter, exec bits, plugin layout)
make build            # Regenerate every derived artifact (skills-codex/, skills-pi/, agents-pi/, AGENTS.md, flat/, hooks)
make check            # build + git diff --exit-code (drift detection вЂ” CI gate)
make fmt              # Auto-format Python and shell files
make skill-evals-fast # Fast paid eval loop: no baseline, no HTML, higher concurrency
```

## Pi Exports

Pi output is generated, not hand-ported. Use:

```bash
make build validate
```

Rules:

- Add `SKILL.pi.md` only when the shared or Codex skill still contains tool names
  or workflows Pi cannot run.
- Use real Pi tools: `read`, `bash`, `edit`, `write`, `ask_user_question`,
  `structured_output`, `todo`, `Agent`, `get_subagent_result`,
  `steer_subagent`, `web_search`, `web_answer`, `web_research`.
- Use `ctx7` or `npx ctx7@latest` for Context7 docs lookup. Do not use
  Context7 MCP in Pi exports.
- Put Pi-only runtime skills under `platforms/pi/skills/`.
- Put Pi-only agents under `platforms/pi/agents/`. Pi agents must be flat `.md`
  files; the filename is the agent name.

Chezmoi deployment should symlink:

```text
~/.pi/agent/skills -> ~/.local/share/cc-thingz/flat/skills-pi
~/.pi/agent/agents -> ~/.local/share/cc-thingz/flat/agents-pi
```

Do not keep copied cc-derived skills or agents in chezmoi after the symlink
migration. Two sources of truth. Twice the rot.

See [Deploy cc-thingz Skills and Agents](docs/pi-skill-export.md) for Pi,
Codex, and Gemini deployment details.

## PR Checklist

- [ ] `make ci` passes
- [ ] Plugin version bumped in both `.claude-plugin/plugin.json` and `.codex-plugin/plugin.json`
- [ ] CHANGELOG.md updated
- [ ] README.md plugin table updated (skill/agent counts)
- [ ] Plugin README.md updated if adding skills, agents, or hooks
- [ ] `make build` run to regenerate every derived artifact (or `make check` confirms a clean tree)
- [ ] Both marketplace files updated if adding/removing plugins
